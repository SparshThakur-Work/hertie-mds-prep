## Key terms
- **Repository (repo)** — a project's folder holding all its files plus the full history of changes
- **Branch** — a separate line of work; lets you make changes without touching the main version
- **Commit** — a saved snapshot of your changes, with a short message describing what changed
- **Pull request (PR)** — a proposal to merge one branch's changes into another, where changes can be reviewed
- **Merge** — combining a branch's changes into the main branch
